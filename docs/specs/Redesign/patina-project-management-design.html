01 — The Six Phases

A design project is a story in six acts.

Every project in Patina follows six phases drawn from real interior design workflow — the same phases Leah walks through with Middlewest clients today. The interface doesn't impose structure; it mirrors the work.

Phase 1
Consultation

Client meeting, questionnaire, site docs, budget & scope

1–2 weeks

Phase 2
Concept

Mood boards, test layouts, 2-3 design options, material palette

2–4 weeks

Phase 3
Refinement

Final selections, detailed specs, client approvals, material orders

4–8 weeks

Phase 4
Procurement

Purchase orders, vendor coordination, delivery tracking, receiving

4–12 weeks

Phase 5
Installation

Delivery coordination, placement, styling, punch list items

1–3 weeks

Phase 6
Walkthrough

Final review, photography, client sign-off, portfolio & review

1–2 weeks

07 — Phase Task View

The work inside each phase.

When a phase is expanded in the project detail, this is the task view. Simple checkboxes with due dates — not a project management app competing with Asana. Tasks can have brief notes attached but no nested subtasks, no dependencies, no Gantt charts. This is a designer's to-do list, not a construction schedule.

09 — Project Closure

Finish strong, build reputation.

When a project reaches the Final Walkthrough phase, the closure flow guides the designer through punch list sign-off, final photography, portfolio capture, and review request. The project isn't archived until these steps are complete — ensuring every project contributes to the designer's portfolio and platform rating.

Design Principle — Projects Are Stories

What makes this different.

Vertical timelines, not Kanban boards. Leah thinks in phases, not swimlanes. The timeline reads top to bottom like the work flows — consultation at the top, walkthrough at the bottom. Each phase has its own color dot so you can spot the current state at a glance.

Tasks are to-dos, not tickets. Simple checkboxes with due dates and one-line notes. No assignees, no priorities, no labels, no comments threads. This is a designer's working list, not a team collaboration tool.

Financials read like an invoice. Budget vs. actual in two columns with Playfair numerals. Sage for under-budget, terracotta for over. Payment milestones show exactly what's been collected and what's outstanding. No pie charts.

Closure earns reputation. The project doesn't just end — it becomes portfolio content, earns a client review, and improves lead quality. Every completed project compounds the designer's value on the platform. The interface makes this visible and motivating.
