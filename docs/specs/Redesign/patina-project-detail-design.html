01 — The Six-Phase Framework

Every project follows the same arc.

Interior design projects progress through six phases. Each phase has a color-coded dot, a defined duration, specific deliverables, a communication cadence, and a set of tasks and decisions. The phase framework is the skeleton that every project detail page hangs on — not a Kanban board, not a Gantt chart, but a vertical timeline that reads like a story.

Consultation

Site visit, questionnaire, room scan, scope definition, service agreement

1–2 weeks

Concept

Mood boards, color palette, style direction, 2–3 design options presented

2–4 weeks

Refinement

Material finalization, product selections, client approvals, spec completion

2–4 weeks

Procurement

Orders placed, vendor coordination, delivery tracking, receiving & inspection

4–12 weeks

Installation

Furniture delivery, placement, styling, accessories, construction oversight

1–3 weeks

Walkthrough

Final review, punch list, photography, documentation handover, celebration

1 week

02 — Client View

What the client sees: their room taking shape.

The client's project page is a progress narrative. Phase dots show where the project stands. A visual progress bar. Key milestones listed beneath. Below, a photo timeline shows the room evolving. Pending decisions surface as interactive cards. Financial summary shows what's been invoiced and what's due. Messages accessible at the bottom. No task lists, no procurement spreadsheets — just the story of their project.

Design Principle — Timelines, Not Kanban

What makes this page work.

Clients see a progress story. Phase dots, a "What's Happening Now" narrative, upcoming milestones, pending decisions as interactive cards, and a simple financial summary. No task checkboxes. No procurement spreadsheets. The room is coming to life — the page should feel like watching it happen.

Decisions live inline. On the designer's view, decisions appear as tagged objects within their phase — pending in golden-hour, resolved in sage, blocking in terracotta. A blocked procurement order shows exactly which decision is holding it. The designer can send a reminder with one click. The client sees decisions as interactive cards at the top of their view.

Financials are transparent. Budget vs. actual in a simple table. Payment milestones with status (paid, due, upcoming). Designer earnings visible only in edit mode. The client sees "paid to date" and "next payment" — not the full procurement spreadsheet. The designer sees everything including line-item-level budget tracking.

One page, no tabs. Phase timeline → procurement orders → financials → activity feed. All in one continuous scroll. No tabbed navigation fragmenting the project across pages. The designer scrolls the same way a client scrolls — top to bottom, beginning to end. The phase timeline IS the navigation — expand any phase to see its details.
