Patina — Product Detail Page Design

The Product PageTwo experiences, one continuous surface

The product detail page serves two audiences from the same URL. Clients see an immersive editorial presentation — story, images, 3D models, materials, maker provenance. Designers see the same page with edit controls overlaid — inline text editing, drag-and-drop gallery management, 3D model upload, and Aesthete intelligence panels. Switch between modes with one toggle. What you edit is what they see.

01 — Page Anatomy

One page, two experiences.

The product detail page is a single continuous scroll with eight content zones. In presentation mode, the client sees an editorial layout — images and story first, specs and intelligence later. In edit mode, the designer sees the same layout with editing controls revealed on hover and a gallery management panel at the top. The toggle between modes takes zero page reloads — it's a CSS-class switch.

Drag, reorder, tag, and upload.

The 3D model gets its own dedicated card with upload, preview, and scale verification. New images drop in via drag-and-drop anywhere on the gallery grid.

04 — Interaction Details

How every element behaves.

Gallery Navigation
Desktop
Arrow keys or click thumbnails. Hover hero for left/right arrows.
Mobile
Swipe hero image. Thumbnail strip scrolls horizontally.
3D Entry
Click "View in 3D" button or tap 3D thumbnail. Viewer replaces hero in place — no page navigate.
AR Launch
From 3D viewer, tap "View in AR". Opens native AR on iOS (USDZ quick-look) or WebXR fallback.
Zoom
Click hero image to open full-screen lightbox. Pinch to zoom on mobile. ESC or swipe down to close.
3D Viewer Controls
Orbit
Click + drag (desktop) or single-finger drag (mobile). 360° horizontal, 180° vertical.
Zoom
Scroll wheel (desktop) or pinch (mobile). Min/max zoom limits set per model.
Details
Preset camera position focusing on joinery, grain detail, or hardware close-up.
Reset
Double-tap or double-click returns to default angle. Smooth 500ms transition.
Lighting
Environment lighting matches Patina warm-white. No user-adjustable lighting in v1.
Edit Mode Gallery
Reorder
Drag handle (⠿) on each thumbnail. Drop to reorder. First position automatically becomes Hero.
Upload
Drag files onto upload zone or click to browse. Multiple files accepted. Auto-detects type by EXIF and aspect ratio.
Tag
Click "Retag" on hover → dropdown: Hero, Lifestyle, Detail, Material, Maker. Tags determine where images appear.
Set Hero
Click "★ Set Hero" on any non-hero image. Swaps positions with current hero automatically.
Remove
Click "Remove" → confirmation toast with 5-second undo. Soft delete initially, permanent after 30 days.
Requirements
Minimum 2000×2000px. JPG/PNG/WebP. Max 5MB per image. System auto-generates responsive sizes.
3D Model Management
Formats
.usdz for iOS AR Quick Look. .glb for Three.js web viewer. Both required for full coverage.
Upload
Drag-and-drop onto dedicated 3D card. Auto-detects format. Validates polygon count and file size.
Scale Check
System compares model bounding box against entered spec dimensions. Flags mismatches with terracotta warning.
Preview
"Preview AR" opens on-device Quick Look. "Preview Web" opens inline Three.js viewer within the edit page.
Texture
PBR workflow required. System checks for baseColor, metallic, roughness, and normal maps. Flags missing maps.
Limits
Max 50k polygons. Max 50MB per model. 4K textures for hero models, 2K for web-optimized.

What makes this page work.

The gallery is an experience, not a grid. Full-bleed hero that fills the viewport. Thumbnail strip for quick navigation. 3D viewer accessible in one tap without leaving the page. Material close-ups as a separate browsable section. Eight images tell a complete visual story: hero angle, alternate view, joinery detail, grain close-up, room context, scale reference, 3D model, and maker workshop.

Edit mode is the same page. No separate admin form. The designer toggles a switch and the same beautiful layout gains editing capabilities — dashed underlines on text, drag handles on images, upload zones where gaps exist. The typography never changes. What Leah edits is precisely what the client will see.

3D models are first-class media. Not a link to a separate viewer — an integrated experience within the product page. The web viewer uses Three.js with PBR materials matching the photography lighting. AR launches natively on iOS via USDZ Quick Look. Both models are managed alongside images in the same gallery grid, with polygon count, texture quality, and scale verification built into the upload flow.

The maker story earns the price. At $4,200, a dining table needs justification beyond specifications. The maker story section — workshop photo, founding year, sourcing philosophy, craft process — transforms a price tag into a narrative. The client isn't buying a table. They're investing in a piece of a 56-year-old workshop tradition that will outlive them. The page structure puts this story where it belongs: after the visual and emotional hook, before the specifications and logistics.
