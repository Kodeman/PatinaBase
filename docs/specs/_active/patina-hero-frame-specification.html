Technical Specification

Time-Based Color Grading

Each time period applies a semi-transparent gradient overlay on top of the captured photo. The overlay uses Core Image filters for GPU-accelerated rendering.

Time Period Greeting Overlay Style Color Temperature
Dawn (5-7) "Early morning." Warm gradient from top-left +800K • Peachy gold
Morning (7-11) "Good morning." Soft warm gradient +400K • Golden
Day (11-14) "Good day." None (neutral) 0K • Neutral
Afternoon (14-18) "Good afternoon." Subtle warm wash +200K • Warm
Evening (18-21) "Good evening." Amber darkening +600K • -15% brightness
Night (21-5) "Good night." Cool dark overlay -200K • -30% brightness

SwiftUI Implementation

enum TimeOfDay {
    case dawn, morning, day, afternoon, evening, night
    
    static func current() -> TimeOfDay {
        let hour = Calendar.current.component(.hour, from: Date())
        switch hour {
        case 5..<7:  return .dawn
        case 7..<11: return .morning
        case 11..<14: return .day
        case 14..<18: return .afternoon
        case 18..<21: return .evening
        default:     return .night
        }
    }
    
    var greeting: String {
        switch self {
        case .dawn:      return "Early morning."
        case .morning:   return "Good morning."
        case .day:       return "Good day."
        case .afternoon: return "Good afternoon."
        case .evening:   return "Good evening."
        case .night:     return "Good night."
        }
    }
}

Technical Specification

Frame Capture During Walk

During the RoomPlan scan, we capture key frames from the AR session to use as hero photos. The best frame is selected automatically, but users can manually override in room settings.

📸 Capture Strategy

  • Sample frames every 2 seconds during scan
  • Store maximum 20 candidate frames per room
  • Include metadata: timestamp, orientation, light estimate
  • Compress to HEIC at 80% quality (~200KB each)

⭐ Scoring Algorithm

  • Sharpness (30%) — Laplacian variance score
  • Brightness (25%) — Histogram distribution
  • Composition (25%) — Feature point spread
  • Stability (20%) — Gyro motion during capture

Frame Scoring Implementation

struct FrameScore {
    let sharpness: Float    // 0-1, Laplacian variance normalized
    let brightness: Float   // 0-1, ideal is 0.5 (mid-range)
    let composition: Float  // 0-1, feature point distribution
    let stability: Float    // 0-1, inverse of motion blur risk
    
    var total: Float {
        (sharpness * 0.30) +
        (brightness * 0.25) +
        (composition * 0.25) +
        (stability * 0.20)
    }
}

func selectHeroFrame(from frames: [CapturedFrame]) -> CapturedFrame? {
    frames
        .map { ($0, scoreFrame($0)) }
        .sorted { $0.1.total > $1.1.total }
        .first?.0
}

Frame Quality Criteria

Criteria Ideal Reject If Detection Method
Sharpness Variance > 500 Variance < 100 Laplacian filter
Brightness Mean 0.35 - 0.65 < 0.15 or > 0.85 Histogram analysis
Motion Blur Gyro < 0.1 rad/s Gyro > 0.5 rad/s CMMotionManager
Feature Spread Features in all quadrants All features clustered ARFrame feature points

Motion Design

Animations & Transitions

⚡ Emergence Pulse

  • Duration: 2.0 seconds
  • Timing: ease-in-out, infinite
  • Scale: 1.0 → 1.25 → 1.0
  • Opacity: 1.0 → 0.5 → 1.0

📱 Room Swipe

  • Duration: 0.4 seconds
  • Timing: spring (damping: 0.85)
  • Parallax: photo moves at 0.8x card speed
  • Pagination: dot width animates 0.2s

⬆️ Companion Pull

  • Interactive: follows finger
  • Detents: 85px, 400px, full screen
  • Spring: response 0.3, damping 0.8
  • Background dims on expand

🌅 Time Transition

  • Triggered on time period change
  • Duration: 0.8 seconds
  • Timing: ease-out
  • Cross-fade between overlays

SwiftUI Animation Definitions

// Emergence pulse
extension Animation {
    static let emergencePulse = Animation
        .easeInOut(duration: 2.0)
        .repeatForever(autoreverses: true)
}

// Room swipe spring
static let roomSwipe = Animation
    .spring(response: 0.4, dampingFraction: 0.85)

// Time overlay transition
static let timeTransition = Animation
    .easeOut(duration: 0.8)

// Companion sheet
static let companionSpring = Animation
    .spring(response: 0.3, dampingFraction: 0.8)

Data Model

Required State & Storage

Core Data Structures

struct Room: Identifiable, Codable {
    let id: UUID
    var name: String
    var heroFrameURL: URL?          // Selected hero photo
    var candidateFrameURLs: [URL]   // All captured frames
    var scannedAt: Date
    var roomPlanData: Data?         // Serialized CapturedRoom
    var savedPieceCount: Int
    var hasEmergence: Bool
}

struct CapturedFrame: Codable {
    let id: UUID
    let imageURL: URL
    let capturedAt: Date
    let qualityScore: Float
    let lightEstimate: Float?       // ARFrame.lightEstimate
    let gyroMotion: Float?          // Motion during capture
}

struct RestStateViewModel: ObservableObject {
    @Published var rooms: [Room] = []
    @Published var currentRoomIndex: Int = 0
    @Published var timeOfDay: TimeOfDay = .current()
    
    var currentRoom: Room? {
        rooms.indices.contains(currentRoomIndex) 
            ? rooms[currentRoomIndex] 
            : nil
    }
    
    func refreshTimeOfDay() {
        timeOfDay = .current()
    }
}
